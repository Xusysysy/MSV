# Plan: 重构动画系统 — 纸条式页码条带模型

## 核心设计

### 条带模型 (Strip Model)

所有页面平铺在一条水平条带上，每个页面宽度 = `pageWidth`（屏幕宽度），高度 = `pageHeight`（等比缩放填满）。

每个页面的 X 位置：
```
offsetX = (pageIndex - currentPage) * pageWidth + transitionOffset
```

- `transitionOffset = 0`：当前页居中，下一页在右侧 `+pageWidth`，上一页在左侧 `-pageWidth`
- `transitionOffset = -pageWidth`：整条向左移一页 → 下一页居中 → 然后 snap 到 0 并 `currentPage += 1`
- `transitionOffset = +pageWidth`：整条向右移一页 → 上一页居中 → 然后 snap 到 0 并 `currentPage -= 1`

### Z-Order（图层顺序）

**页码小的在上层。** 在 Compose 中，后渲染的孩子在上层 → 按页码**降序**迭代（大→小），让页码小的最后渲染。

翻下一页（1→2）：页1（上层）滑左，露出页2（下层）
翻上一页（2→1）：页1（上层）从左侧滑右，遮住页2（下层）

### 渲染范围

`currentPage-3 .. currentPage+3`（7页），边界裁剪到 `0..pageCount-1`。

### 页面大小

所有页面统一大小：`pageWidth × pageHeight`，其中：
- `pageWidth = viewportWidth`（最大占满屏幕宽）
- `pageHeight = pageWidth / pageAspectRatio`（按 PDF 原始比例等比缩放）
- 图片 `ContentScale.FillBounds` 填满固定尺寸框

## 文件修改

### 1. `data/model/ViewerState.kt`
- 新增 `pageUris: Map<Int, Uri> = emptyMap()` — 页码→渲染图片 URI
- 新增 `pageWidth: Int = 0`, `pageHeight: Int = 0` — 统一页面尺寸
- 删除 `prevPageUri`, `nextPageUri`, `currentPageUri`（用 pageUris 替代）

### 2. `viewmodel/ViewerViewModel.kt`
- 重写 `preloadAdjacentPages()` → 渲染 `currentPage±3` 到 `pageUris`
- `goToPage()` 更新 `pageUris[currentPage]` 和触发预加载
- `handleFilesSelected()` / `openPdf()` / `openImages()` 初始化 pageWidth/pageHeight
- 渲染时获取 PDF 页面原始宽高比，计算统一 pageWidth/pageHeight
- 缩略图调用 `getThumbnailUri(pageIndex)` 保持不变

### 3. `ui/components/Stage.kt` — 完全重写

**接口简化：**
```kotlin
fun Stage(
    pageUris: Map<Int, Uri>,
    currentPage: Int,
    pageCount: Int,
    pageWidth: Int,
    onPageFlip: (Int) -> Unit,  // dir: -1=prev, +1=next
    ...
)
```

**布局：**
- `forEachIndexed` 遍历 `(currentPage-3..currentPage+3)`，降序排列
- 每页 `offset { IntOffset(offsetX, 0) }`，`size(pageWidth.dp, pageHeight.dp)`
- `ContentScale.FillBounds` 填满统一尺寸

**手势：**
- 单个 `pointerInput(Unit)` 块，协程手动处理
- `detectHorizontalDragGestures` 拖拽跟踪：`transition.snapTo(dragOffset)`
- `detectTapGestures` 点击翻转：左半区→prev，右半区→next
- 缩放/平移单独 `pointerInput(isZoomed)`

**动画：**
- 拖拽结束 → 阈值判断 → `animateTo(±pageWidth)` → `snapTo(0)` → `onPageFlip(dir)`
- 点击翻转 → `onPageFlip(dir)` 直接触发，Stage 播放滑出动画后响应

**纸叠视觉：**
- 保留右侧纸叠边缘指示器（基于 `currentPageCount - currentPage` 剩余页数）

### 4. `ui/screen/ViewerScreen.kt`
- 更新 Stage 调用签名，传入 `pageUris`, `pageWidth`, `pageHeight`
- 删除不需要的回调

## 数据流

```
用户拖拽 → Stage.animateTo(±pageWidth) → snapTo(0) → onPageFlip(dir)
  → ViewerViewModel.goToPage(currentPage + dir)
    → 更新 currentPage, pageUris
    → preloadAdjacentPages() 异步渲染 ±3 页
  → Stage 重组 → 新 currentPage 居中，7 页布局更新
```
